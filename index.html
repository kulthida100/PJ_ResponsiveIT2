สำหรับผู้เริ่มต้น

เรียนรู้การใช้ Git และ GitHub แบบง่าย ๆ

Git คือระบบจัดการเวอร์ชันของไฟล์ส่วนตัวและทีมงาน ส่วน GitHub คือแพลตฟอร์มสำหรับเก็บโค้ดบนคลาวด์และทำงานร่วมกันได้อย่างสะดวก

เริ่มใช้งานเลย

สิ่งที่คุณจะได้เรียนรู้

  • Git คืออะไรและทำไมต้องใช้
  • คำสั่งพื้นฐานที่ควรรู้
  • วิธีอัปโหลดโปรเจกต์ขึ้น GitHub

Git กับ GitHub ต่างกันอย่างไร?

Git

Git เป็นเครื่องมือจัดการเวอร์ชันของโค้ดที่ช่วยติดตามความเปลี่ยนแปลงและย้อนกลับได้

GitHub

GitHub เป็นบริการเว็บสำหรับเก็บ repository, แชร์โค้ดและร่วมมือกับคนอื่น

Why use it?

ช่วยให้ทำงานเป็นทีมได้ดีขึ้น ปลอดภัยกว่า และไม่ต้องกลัวการลบไฟล์สำคัญ

คำสั่งพื้นฐานที่ควรรู้

1. สร้าง repository ใหม่

git init

เริ่มติดตามโฟลเดอร์ปัจจุบันเป็น Git repository

2. เพิ่มไฟล์เข้า staging area

git add .

เตรียมไฟล์ที่ต้องการบันทึกการเปลี่ยนแปลง

3. บันทึกความเปลี่ยนแปลง

git commit -m "เพิ่มหน้าแรก"

บันทึก snapshot ของงานในปัจจุบัน

4. ส่งขึ้น GitHub

git push origin main

อัปโหลดโค้ดไปยังเซิร์ฟเวอร์ของ GitHub

Workflow แบบง่าย ๆ

  1. สร้างหรือ clone repository จาก GitHub
  2. แก้ไขไฟล์และบันทึกงาน
  3. ใช้คำสั่ง git add และ git commit
  4. ส่งงานด้วย git push
  5. ตรวจสอบความเปลี่ยนแปลงบน GitHub
git clone https://github.com/username/repo.git
cd repo
git checkout -b feature/new-page
git add .
git commit -m "เพิ่มหน้าใหม่"
git push origin feature/new-page
